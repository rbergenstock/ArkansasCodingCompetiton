Competition Strategy

⏱ Time Management (3 Hours)

You have 180 minutes. Plan deliberately.

Recommended: Spend 5 min scanning ALL challenges. Sort by difficulty. Assign team members by strength. Easy first — bank points early.

Past competitions: 7 challenges from 4 pts to 28 pts. Get easy wins in 20 min, then divide and conquer.

🖥 One-Computer Workflow

Only one student on input devices at a time — biggest tactical factor.

Paper-first method (proven by past winners): Each member works a separate problem on paper. Write pseudocode and actual code. When it's your turn at the keyboard, you're just typing. Rotate efficiently.

📋 Read Everything Twice

Output format is extremely specific. Spelling, case, spacing, punctuation all matter.

Common traps: Missing colon/space in output. Case sensitivity. Not handling edge cases (missing files, invalid input). Read the rubric for partial credit breakdown.

🎯 Point Strategy

4 pts — Print team info (free, do first) · 8-14 pts — String manipulation, encoding, validation · 16-22 pts — File I/O, JSON/CSV, multi-step algorithms · 22-28 pts — Complex ciphers, multi-file traversal

Partial credit is real! Even getting the prompt/input handling (usually 2 pts) is worth it. Never leave a challenge blank.

🧠 Key Topics

Strings: substring, replace, remove, case, ASCII · File I/O: JSON, CSV, text — existence checks, read, write, append · Encoding: Base conversion, Caesar variants, character shifting · Data structures: Arrays, HashMaps · Validation: Date formats, re-prompt loops · MC concepts: Design patterns, Big-O, CIA triad, SQL, OOP, bitwise, base conversion
